第二章 模型评估与选择

过拟合overfitting

学习器把训练样本本身的一些特点当作了所有潜在样本都会具有的一般性质，导致泛化性能下降

因素：学习能力过于强大

解决：过拟合无法避免，能“缓解”，各个算法有不同的解决方法

欠拟合underfitting

相对于过拟合，对训练样本的一般性质尚未学好

因素：学习能力低下

解决：增加训练轮数

训练集

测试集应该尽可能与训练集互斥

划分方法：

  1. 留出法hold-out

    直接将数据集，根据比例，划分成两个互斥的集合，一个做训练集，一个做测试集

  2. 交叉验证法cross validation

    将数据集划分为k个大小相同的互斥子集

    进行k次训练，每次取k-1个子集作为训练集，余下的做测试集

    最终取k次训练结果的平均值

    又称“k折交叉验证” k-fold cross validation

    k最常取值为10

    同时，由于将数据集划分为子集有多种划分方式，为了减少因样本划分带来的误差，k折交叉验证通常要使用不同的划分方式重复p次

    例如最常用的“10次10折交叉验证”

  3. 自助法 bootstrapping

    以自助采样法(bootstrapping sampling)为基础

    每次随机从数据集中挑选一个样本，获得其拷贝，然后将样本放回数据集

    这个过程重复m次后，得到了一个包含m个数据的数据集

    虽然有可能一些数据重复出现，一些数据没出现

    适用于数据集较小或难以划分训练测试集的情形

    也可以从初始数据集，产生多个不同的训练集

调参 parameter tuning

算法的参数称为“超参数”hypterparameter，通常不多

模型的参数，称为参数parameter，通常很多

在模型选择完成后，学习算法和参数配置已经选定之后，我们应该使用数据集重新训练模型，使用所有的样本，从而形成最终提交给用户的模型

在研究对比不同算法的泛化性能时

使用在测试集上的判别效果来估计模型的实际泛化能力

而把训练数据再分为训练集和验证集，基于验证集来进行该算法的模型选择和调参

性能度量

混淆矩阵confusion matrix 预测结果 预测结果
真实情况 正例 反例
正例 TP(真正例) FN(假反例)
反例 FP(假正例) TN(真反例)

查准率（准确率）precision

查全率（召回率）recall

P-R曲线 纵轴P，横轴R，调整阈值的大小画出

不同的学习器，包住的面积越大，性能越好

或者平衡点，Break-Even Point，在45°角的线上，曲线的值，越大越好

或者F1得分

当我们需要对查全/差准有一定偏向的时候，我们使用F1的一般形式

更有，当我们拥有n个二分类混淆矩阵的时候，我们有两种方法进行综合考察

  • 一是，在各个混淆矩阵中计算出查准率和查全率后，得出平均查准率查全率，然后使用平均查准率查全率计算出F1得分，称为“宏F1”
  • 二是，统计各个混淆矩阵中的TP、FP、TN、FN的平均值，然后再计算查准率查全率，然后再计算F1得分，称为“微F1”

ROC和AUC

ROC 受试者工作特征Receiver Operating Characteristic

AUC Area Under Curve

与P-R曲线类似，ROC也是用两个值作为横纵坐标

真正例率True Positive Rate

假正例率False Positive Rate

ROC以TPR为纵轴，以FPR为横轴，也是通过设定不同的阈值画线

第三章 线性模型

基本形式

线性模型形式简单，易于建模，拥有很好的可解释性comprehensibility

非线性模型可以在线性模型的基础上引入层级结构和高维映射而构成

线性回归 linear regression

对于离散属性，若属性值间存在着序(order)的关系，则通过连续化转化为连续值

例如身高属性的高、中、低 可以转化为{1.0,0.5,0.0}

但是如果是例如黄瓜、西瓜、南瓜这类转换成(0,0,1),(0,1,0),(1,0,0)会不恰当引入序关系，导致后续处理如计算距离造成误导

均方误差（取差平方再求和）有非常好的几何意义，例如欧氏距离Euclidean distance

基于均方误差最小化来进行模型求解的方法称为最小二乘法least square method

当属性大于1时，称为多元线性回归multivariate linear regression

对数几率回归logistic regression

对于二分类任务，线性回归模型产生的是实值，我们需要将实值转换为0/1值

对数几率函数，是一种sigmoid函数（形似s的函数），将z值转化为一个接近0或1的y值

线性判别分析 linear discriminant analysis

给定训练集，设法将训练集投影到一条直线上，使得同类的投影点尽可能解禁，异类的投影点尽可能远离。当一个新样本进来的时候，将其投影到直线上，根据投影点位置进行分类。

LDA通常也用来有监督的降维

多分类学习

一对一One vs. One

当有N个类别，那么我们队这N个类别两两配对，产生N(N-1)个二分类任务，然后产生N(N-1)个分类器，最终，通过所有的分类器预测最多的类别作为最终预测结果。

一对多One vs. All

每次将一个类别的样例作为正例，其他的都作为反例，从而训练N个分类器，然后通过所有的分类器，置信度最高的作为最终预测结果

多对多Many vs. Many

纠错输出码Error Correcting Output Codes

编码：

将N个类别做M次划分，一部分正类一部分反类（二元码，三元码还会划分停用类），形成M个训练集，M个分类器

解码：

M个分类器分别对测试样本进行预测，这些预测标记组成一个编码，与每个类别各自的编码进行比较，返回距离最小的为预测结果

类别不平衡class-imbalance

  • 阈值移动threshold-moving

    在置信值与阈值比较之前，对置信值乘以一个偏差值，偏差值由真实几率决定

    但是真实几率往往不能观测出来

  • 欠采样undersampling

    去除一些反例，使得正反比例相近

  • 过采样oversampling

    增加一些正例，使比例接近
